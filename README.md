# perShop 🛍

فروشگاه استاتیک با مدیریت محصولات از طریق بات تلگرام.

## ساختار پروژه
```
perShop/
├── bot/          ← بات تلگرام (ادمین‌پنل)
└── site/         ← سایت استاتیک (روی GitHub Pages میزبانی می‌شه)
```

## مرحله ۱: ساخت ریپوی گیت‌هاب
1. یک ریپو جدید بساز، مثلاً `SAY4s/perShop`.
2. کل پوشه `perShop/` رو داخلش push کن (هم `bot/` هم `site/`).
3. در تنظیمات ریپو → Pages → سورس رو روی شاخه `main` و پوشه `/site` بگذار (یا اگه گیت‌هاب اجازه نداد پوشه‌ی فرعی رو انتخاب کنی، محتوای `site/` رو به یک ریپوی جدا یا شاخه `gh-pages` منتقل کن).

## مرحله ۲: ساخت بات تلگرام
1. به [@BotFather](https://t.me/BotFather) پیام بده، با `/newbot` یک بات بساز.
2. توکن رو کپی کن.
3. یوزرنیم بات (مثلاً `perShop_bot`) رو در دو جا بگذار:
   - `bot/.env` → `BOT_USERNAME`
   - `site/script.js` → متغیر `BOT_USERNAME` در بالای فایل
4. آدرس GitHub Pages سایتت (مثلاً `https://SAY4s.github.io/perShop`) رو در `bot/.env` بگذار:
   - `SITE_URL=https://SAY4s.github.io/perShop`

## مرحله ۳: ساخت GitHub Token
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. یک توکن با اسکوپ `repo` بساز.
3. در `bot/.env` بگذار: `GITHUB_TOKEN=...`

## مرحله ۴: گرفتن آیدی عددی ادمین
1. به [@userinfobot](https://t.me/userinfobot) پیام بده تا آیدی عددیت رو بگیری.
2. در `bot/.env`: `ADMIN_IDS=123456789` (برای چند ادمین با کاما جدا کن)

## مرحله ۵: تکمیل فایل `.env`
فایل `bot/.env.example` رو کپی کن به `bot/.env` و مقادیر واقعی رو بگذار:
```
BOT_TOKEN=...
ADMIN_IDS=...
GITHUB_TOKEN=...
GITHUB_OWNER=SAY4s
GITHUB_REPO=perShop
GITHUB_BRANCH=main
PRODUCTS_PATH=site/data/products.json
ASSETS_PATH=site/assets
BOT_USERNAME=perShop_bot
SITE_URL=https://SAY4s.github.io/perShop
```

## مرحله ۶: اجرای بات
روی سرور/کامپیوتر همیشه‌روشن:
```bash
cd bot
pip install -r requirements.txt
python main.py
```

## مرحله ۷: تست
1. به ادمین‌پنل بات `/add` بزن و یک محصول بساز (عنوان، توضیح، قیمت، دسته، موجودی، عکس).
2. چک کن که در ریپوی گیت‌هاب کامیت جدید زده شده (در `site/data/products.json` و `site/assets/`).
3. آدرس GitHub Pages رو باز کن — محصول باید دیده شه (ممکنه چند دقیقه طول بکشه تا Pages آپدیت شه).
4. روی دکمه «سفارش از طریق تلگرام» کلیک کن — باید با دیپ‌لینک به بات بری و فلوی سفارش شروع شه.

## دستورات ادمین در بات
| دستور | کار |
|---|---|
| `/add` | افزودن محصول جدید (فلوی مرحله‌ای) |
| `/list` | لیست محصولات با دکمه‌های ویرایش/حذف |
| `/shop` | باز کردن فروشگاه به‌صورت Mini App (برای تست) |
| `/help` | راهنما |

## 📱 Mini App (باز شدن سایت داخل تلگرام)
وقتی بات اجرا می‌شه، خودش یک دکمه ثابت 🛍 کنار باکس پیام (Menu Button) نصب می‌کنه که با کلیک، همین سایت رو **بدون خروج از تلگرام** باز می‌کنه. این یعنی مشتری از همون چت بات می‌تونه فروشگاه رو ببینه و گردش کنه.

نکات لازم برای این‌که Mini App درست کار کنه:
- آدرس `SITE_URL` در `.env` باید HTTPS باشه (GitHub Pages به‌صورت پیش‌فرض HTTPS می‌ده، پس مشکلی نیست).
- اگه `SITE_URL` رو خالی بگذاری، دکمه‌ی منو نصب نمی‌شه ولی بقیه‌ی بات عادی کار می‌کنه.
- سایت از `Telegram.WebApp` API استفاده می‌کنه (تشخیص خودکار، اگه خارج از تلگرام هم باز شه مشکلی نداره) تا:
  - با تم رنگی موبایل/دسکتاپ سازگار باز شه (`expand()`)
  - دکمه «سفارش» مستقیم به چت بات (نه مرورگر خارجی) برسونه
  - لرزش لطیف (Haptic Feedback) موقع تَپ روی دکمه‌ها بده

اگه می‌خوای دکمه‌ی منو رو دستی هم در BotFather تنظیم کنی (اختیاری، چون بات خودش این کارو می‌کنه):
`/mybots` → انتخاب بات → `Bot Settings` → `Menu Button` → آدرس و متن دکمه رو بگذار.

## نکات مهم
- هر بار که ادمین تغییری اعمال کنه، یک کامیت جدید روی گیت‌هاب زده می‌شه؛ سایت با تأخیر کوتاهی (چند دقیقه، به‌خاطر کش GitHub Pages) آپدیت می‌شه.
- `script.js` با `?v=timestamp` کش مرورگر رو دور می‌زنه، اما کش خود GitHub Pages CDN ممکنه چند دقیقه طول بکشه.
- اگه چند ادمین هم‌زمان محصول اضافه/ویرایش کنن، کد هر بار قبل از نوشتن SHA تازه می‌گیره تا تعارض پیش نیاد.
- مدل فعلی خرید «هماهنگی دستی» است: مشتری از سایت به بات می‌ره، توضیح/تماس می‌ده، بات به همه‌ی ادمین‌ها پیام رو فوروارد می‌کنه و خرید نهایی دستی پیگیری می‌شه. اگه بعداً خواستی درگاه پرداخت آنلاین اضافه کنی، باید یک وب‌هوک یا بک‌اند کوچک جدا بسازیم.

## مراحل بعدی پیشنهادی (وقتی خواستی توسعه بدیم)
- افزودن دستور `/stats` برای آمار فروش
- لاگ تغییرات در یک کانال تلگرام جدا (`LOG_CHAT_ID`)
- صفحه جزئیات محصول با URL مجزا (برای اشتراک‌گذاری مستقیم لینک محصول)
- پشتیبانی از چند عکس برای هر محصول
